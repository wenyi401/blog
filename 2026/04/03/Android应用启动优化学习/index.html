前言

Android 应用的启动性能是用户体验的重要组成部分。一个启动缓慢的应用不仅会让用户感到烦躁，还可能导致用户放弃使用。

一、启动类型

冷启动

从头开始创建应用进程：

  1. 加载并启动应用
  2. 创建应用进程
  3. 创建 Application 对象
  4. 启动主线程
  5. 创建 MainActivity
  6. 布局、绘制

热启动

应用进程已存在，只需将 Activity 带到前台。

温启动

应用进程存在，但需要重新创建 Activity。

五、优化清单

Application 优化

优化项 描述
✅ 延迟初始化 非必要 SDK 异步初始化
✅ 使用 App Startup 统一管理初始化
✅ 避免耗时操作 不执行 I/O 操作
✅ 减少锁争用 避免同步锁

Activity 优化

优化项 描述
✅ 简化布局 减少布局层级
✅ 延迟加载 非必要 View 延迟加载
✅ 使用 ViewStub 按需加载
✅ 避免主线程 I/O 移至子线程

资源优化

优化项 描述
✅ 压缩图片 使用 WebP
✅ 移除无用资源 shrinkResources
✅ 使用矢量图 VectorDrawable

七、最佳实践

  1. 异步初始化：所有非必要的初始化移至子线程
  2. 使用 App Startup：统一管理组件初始化
  3. 延迟加载：按需加载非必要 UI
  4. 优化布局：减少布局层级
  5. 持续监控：使用 Macrobenchmark 持续测量
