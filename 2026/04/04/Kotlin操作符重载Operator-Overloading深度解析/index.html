前言

Kotlin 允许为类型提供预定义操作符的自定义实现。这些操作符有预定义的符号表示（如 + 或 *）和优先级。

二、一元操作符

前缀操作符

表达式 转换为
+a a.unaryPlus()
-a a.unaryMinus()
!a a.not()

递增和递减

表达式 转换为
a++ a.inc()
a– a.dec()

三、二元操作符

算术运算符

表达式 转换为
a + b a.plus(b)
a - b a.minus(b)
a * b a.times(b)
a / b a.div(b)
a % b a.rem(b)
a..b a.rangeTo(b)
a..<b a.rangeUntil(b)

in 操作符

表达式 转换为
a in b b.contains(a)
a !in b !b.contains(a)

四、索引访问操作符

表达式 转换为
a[i] a.get(i)
a[i, j] a.get(i, j)
a[i] = b a.set(i, b)
a[i, j] = b a.set(i, j, b)

五、invoke 操作符

表达式 转换为
a() a.invoke()
a(i) a.invoke(i)
a(i, j) a.invoke(i, j)

六、增强赋值

表达式 转换为
a += b a.plusAssign(b)
a -= b a.minusAssign(b)
a *= b a.timesAssign(b)
a /= b a.divAssign(b)
a %= b a.remAssign(b)

七、相等和比较操作符

相等操作符

表达式 转换为
a == b a?.equals(b) ?: (b === null)
a != b !(a?.equals(b) ?: (b === null))

比较操作符

表达式 转换为
a > b a.compareTo(b) > 0
a < b a.compareTo(b) < 0
a >= b a.compareTo(b) >= 0
a <= b a.compareTo(b) <= 0

十、常见问题

问题 1：歧义错误

原因：同时定义了 plus() 和 plusAssign()。

解决方案：只保留一个，或确保返回类型正确。

问题 2：类型不匹配

原因：操作符返回类型与预期不符。

解决方案：确保返回类型符合操作符语义。

问题 3：性能问题

原因：操作符创建过多临时对象。

解决方案：使用可变类或基本类型。
